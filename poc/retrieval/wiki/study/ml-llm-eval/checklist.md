⏺ Block 1: Stats Foundations

  ---
  Lesson 1: P-value (Q001 — asked 3x on Glassdoor)

  What it is: The probability of observing data as extreme as (or more extreme
  than) what you got, assuming the null hypothesis is true.

  Key points to nail in interview:
  - It is NOT the probability that H0 is true
  - It is NOT the probability your result happened by chance
  - Low p-value (< 0.05) = "this data would be very unlikely if H0 were true" →
  reject H0
  - Example: p = 0.03 means "if there's truly no effect, we'd see data this
  extreme only 3% of the time"

  Common follow-up trap: "So p = 0.03 means 3% chance the null is true?" → No.
  P-value is about the data given H0, not H0 given the data.

  ---
  Lesson 2: Mean vs Median (Q002 — asked 3x)

  Use mean when: Data is roughly symmetric, no extreme outliers. You want a
  measure that uses all data points.

  Use median when: Data is skewed or has outliers (income, home prices,
  latency). Robust to extremes.

  Interview gold: "Mean of [1,2,3,4,100] = 22. Median = 3. Which better
  represents the typical value?" → Median.

  Bonus: Mean is useful when you care about totals (e.g., average revenue per
  user × N users = total revenue). Median can't do that.

  ---
  Lesson 3: Standard Error of the Mean (Q003 — asked 3x)

  Formula: SE = σ / √n

  What it is: How much the sample mean would vary across repeated samples. It
  measures precision of your estimate, not variability of the data.

  Key distinction:
  - Standard deviation → spread of the data
  - Standard error → spread of the sample mean (how precise your estimate is)

  Why √n: More data → more precise → SE shrinks. To cut SE in half, you need 4x
  the data.

  ---
  Lesson 4: Margin of Error & Sample Size (Q004 — asked 2x)

  The question: Margin of error is 3 with sample size n. How many samples to
  make it 0.3?

  Answer: Margin of error ∝ 1/√n. To reduce MOE by 10x (from 3 to 0.3), you need
   100x the sample size. 100n total samples, so 99n more.

  ---
  Lesson 5: Confidence Intervals (Q020, Q021)

  What a 95% CI means: If we repeated this experiment many times, 95% of the
  intervals we construct would contain the true parameter.

  NOT: "95% probability the true value is in this interval." (The true value is
  fixed; the interval is random.)

  Width of CI: 2 × z × SE = 2 × z × (σ/√n). Wider with more confidence (99% >
  95%), smaller with more data.

  ---
  Lesson 6: Simpson's Paradox (Q023)

  Q: Can two group means both rise, but the pooled mean decrease?

  Yes. If the mix of groups shifts toward the lower-mean group. Classic example:
   both departments improve admission rates, but overall rate drops because more
   applicants shifted to the harder department.

  ---
  Lesson 7: Bootstrap (Q015, Q016)

  What: Resample with replacement from your data to estimate the sampling
  distribution of a statistic.

  Does it increase sample size? No. It estimates uncertainty of your statistic.
  It doesn't create new information — it reuses what you have.

  How to implement:
  1. Draw n samples with replacement from your data
  2. Compute statistic (mean, median, whatever)
  3. Repeat B times (e.g., 10,000)
  4. The distribution of those B statistics ≈ sampling distribution

⏺ You're on the right track with "2 throws." Here's the answer, then the

  ---
  Answer: Von Neumann's trick

  Flip the biased coin twice. There are 4 outcomes:

  ┌────────┬────────┬─────────────┐
  │ Flip 1 │ Flip 2 │ Probability │
  ├────────┼────────┼─────────────┤
  │ H      │ H      │ p * p       │
  ├────────┼────────┼─────────────┤
  │ H      │ T      │ p(1-p)      │
  ├────────┼────────┼─────────────┤
  │ T      │ H      │ (1-p)p      │
  ├────────┼────────┼─────────────┤
  │ T      │ T      │ (1-p)^2     │
  └────────┴────────┴─────────────┘

  HT and TH have equal probability regardless of p. So:

  - If HT → call it "Heads"
  - If TH → call it "Tails"
  - If HH or TT → throw away, repeat

  Done. Fair coin from any unfair coin.

  ---
  The Universal Thinking Tool: Symmetry Hunting

  Both this question and Q6 (elevator) are the same type: use symmetry to count 
  or equalize outcomes.

  3-step method for any probability question:

  1. Define the sample space — what are ALL possible outcomes? (Elevator: each
  person picks independently → 4^4. Coin: list all 2-flip combos)
  2. Find the symmetry — which outcomes have equal probability by construction?
  (Elevator: all 4^4 equally likely. Coin: HT and TH are symmetric)
  3. Count or condition — count favorable/total (elevator), or condition on the
  symmetric subset (coin)

  Your mistake on Q6: you jumped to 4! as the total instead of first asking "how
   does each person choose?" → independently → 4^4.

  Your instinct on Q7: "some condition... conditional probability that cancels
  out p" — that was exactly right. You were looking for symmetry but didn't
  write out the 2x2 table to see it.

  Rule: when stuck, write out the full sample space. It's never more than a few
  lines for interview questions, and the answer usually jumps out.

  Q12 (Transformers / LLM Eval): What is a Transformer? Explain it.

❯ Transformer is architecutre that helps with feature extraction from text.     
this works using the attention mechanism key value formulation. it measures the 
 correlations between words in the same doc and are foundation of modern llms   

⏺ Decent high-level answer. For Google, you need a bit more structure. Here's
  what they want:

  Transformer in 30 seconds:

  1. Input: tokens → embeddings + positional encoding (because no recurrence,
  order must be injected)
  2. Self-attention: each token computes Query, Key, Value vectors. Attention
  score = softmax(QK^T / √d) × V. This lets each token "look at" every other
  token and decide what's relevant.
  3. Multi-head: run attention in parallel with different learned projections →
  captures different types of relationships
  4. Feed-forward: after attention, each token passes through a small neural net
  5. Stack: repeat this block N times. Add residual connections + layer norm.
  6. Original paper: encoder-decoder architecture. Modern LLMs (GPT) use
  decoder-only.
